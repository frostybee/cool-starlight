### Slide 3: Hashing Passwords


## ğŸ” Hashing Passwords with bcrypt

Never store plain-text passwords!

- Use PHP's `password_hash()` and `password_verify()`

```php
// Registration
$hash = password_hash($password, PASSWORD_DEFAULT);

    // Success
}
```

### Slide 6: Role-Based Access Control (RBAC)

## ğŸ”‘ Role-Based Access Control (RBAC)

- Users have assigned roles: `admin`, `customer`, `guest`
- Define permissions based on role

**Permissions Example:**
| Role      | Action                |
|-----------|------------------------|
| Admin     | Manage all products     |
| Customer  | View & buy products     |
| Guest     | Browse products only    |

- Use role checks in your controllers/pages


---

### Slide 7: Admin vs Customer Permissions

## ğŸ‘¥ Admin vs Customer Permissions

- **Admin**
  - Can CRUD products
  - View and manage orders
  - Access dashboard

- **Customer**
  - Can browse and purchase
  - View order history

- Access Logic:

```php
if ($_SESSION['role'] === 'admin') {
    // show admin panel
} else {
    // show customer content
}
```

### Slide 10: Preventing XSS Attacks

## ğŸ§¼ Preventing Cross-Site Scripting (XSS)

XSS = Injecting malicious scripts into pages

**How to prevent:**
- Escape output using `htmlspecialchars()`
```php

```

- Sanitize user inputs on forms
- Use Content Security Policy (CSP) headers
